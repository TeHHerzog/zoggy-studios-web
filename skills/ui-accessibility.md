# UI and Accessibility Skill

## Goal

Produce clear, responsive, keyboard-friendly public pages before adding decorative complexity.

## Required checks

- one `h1` per page
- logical heading order
- semantic `header`, `nav`, `main`, and `footer`
- skip-to-content link
- visible keyboard focus
- meaningful link and button text
- sufficient contrast
- correct `lang` attribute
- usable at 200 percent browser zoom
- no hover-only controls
- no motion required to understand content
- honor `prefers-reduced-motion`
- useful alt text for meaningful images
- empty alt text for decorative images

## Responsive checks

Review around:

```text
320px
375px
768px
1024px
1440px
```

Look for:

- horizontal overflow
- clipped headings
- unreadable line length
- cramped buttons
- broken navigation
- overlapping content
- inaccessible focus states

## Design priority

1. readability
2. navigation clarity
3. accessibility
4. performance
5. visual atmosphere
